[1] 테스트 프로젝트와 단위 테스트
	단위 테스트 : 특정 클래스의 메서드에 있는 논리 오류를 빠르게 찾을 수 있도록 도와주는 기능이다.

	.NET 스텐다드 프로젝트는 User Interface가 없다.
	모양을 꾸미는 것이 아니라 API를 담고 있는 것이다.
	이러한 API의 오류를 잡아내기 위한 테스트를 수행해야한다.

	테스트는 전체 테스트가 아니라 단위로 나누어 테스트를 수행한다.
	흔히 메서드 단위로 테스트를 수행한다.
	
	Visual Studio에는 많은 테스트 도구들이 내장되어 있따.
	이번에는 여러 테스트 프로젝트 중 하나인 MSTest를 사용하여 클래스 라이브러리에 작성된 C# 코드에 대한 테스트를 진행할 수 있다.

	하나의 클래스 라이브러리 안에 있는 기능들을 간단하게 테스트를 수행한다.
	API 개발 후 단위 테스트 단계가 프로그래밍 기본 절차이다.

[2] 자동 테스트 (Automated Test)
	특정한 API에 대한 테스트를 작성해 놓으면, 필요할 때에 자동으로 테스트를 수행할 수 있다.
	이러한 자동 테스트를 진행하게 되면 다음과 같은 장점을 가진다.
	- 에러를 쉽게 찾고 고칠 수 있다.
	- 직접 하나씩 테스트하는 것 보다 아주 빠르게 많은 수의 테스트를 진행한다.
	- 작성한 코드에 대한 검증을 할 수 있다.

[3] MSTest
	MSTest는 Visual Studio에 기본적으로 내장되어 있는 테스트 프로젝트이다.
	C#에서는 다음과 같은 단위 테스트 프로젝트를 사용할 수 있다.
	- MSTest : 마이크로소프트 공식 단위 테스트 프로젝트
	- NUnit : JUnit 영향을 받은 오픈 소스 단위 테스트 프로젝트
	- xUnit : 닷넷 전용 오픈 소스 단위 테스트 프로젝트

	1) Assert 클래스 주요 API
	 Assert.IsTrue() : 매개변수 값이 true이면 테스트 통과
	 Assert.IsNull() : 매개변수 값이 null이면 테스트 통과
	 Assert.AreEqual() : 매개변수 값 2개가 같으면 테스트 통과
	 Assert.AreNotEqual() : 매개변수 값 2개가 다르면 테스트 통과
