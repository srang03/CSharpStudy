[1] 인메모리 데이터베이스
	현대적인 응용프로그램들은 대부분 앱에서 사용하는 데이터를 데이터베이스에 저장한다.
	업무용 앱에서 DBMS를 사용하지 않는 앱은 거의 없다.
	C#에서는 이러한 DBMS와 데이터를 주고 받는 클래스를 다수 제공한다.
	이러한 클래스들의 집합을 다른 말로 ADO.NET라고 한다.
	C#에서는 DBMS와 데이터를 주고 받는 클래스를 다수 제공한다.
	DBMS 관련 클래스들의 집합을 ADO.NET이라고 한다.

	CRUD : 컬렉션, 파일, 데이터페이스 등의 저장소에 데이터를 저장, 조회, 수정, 삭제하는 작업이다. (크러드 작업)
	업무용 앱의 대부분의 로직은 이러한 CRUD 작업으로 표현할 수 있다.

[2] CRUD 네이밍
	- CRUD 관련 네이밍
	
	- BREAD (Browse, Read, Edit, Add, Delete)
	- BREAD SHOP(Search Has Ordering Paging)
		Browse : 단일 데이터 
		Read : 여러 데이터
		Has : Count

[3] 리포지토리 패턴 (Repository Pattern)
	저장소에서 데이터를 읽어오는 것은 리포지토리 클래스
	- 모델 클래스 : 데이터의 구조
	- 리포지토리 클래스 : 저장소와 관련된 데이터 입출력 관련 기능
	- 컨텍스트 클래스 : 모델과 리포지토리를 묶어서 실제 업무 형태로 관리해주는 것을 컨텍스트 클래스라고 한다.
						실제 데이터 공간을 리파지토리 클래스를 사용해서 입출력하고, 
						리파지토리 클래스를 호출해서 비즈니스 로직을 컨텍스트 클래스에 담아서 사용한다.
	비즈니스 애플리케이션에서 사용하는 코드 패턴
