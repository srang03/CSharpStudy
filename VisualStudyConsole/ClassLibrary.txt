[1] 클래스 라이브러리(.NET Standard)
	클래스 라이브러리를 만들어 공통코드를 모아두는 방법
	여러 프로젝트에서 공통적으로 사용할 기능들을 클래스 라이브러리 프로젝트에 모아서 관리할 수 있다.

	클래스 라이브러리 (.NET Standard)로 프로젝트를 만들게 되면, .dll 파일로 클래스들이 압축되어 최종 결과물을 만들어낸다.
	.NET 생태계의 모든 환경에서 사용할 수 있는 나만의 패키지, 라이브러리를 만들고 싶을 때 클래스 라이브러리를 사용한다.

[2] 닷넷 스탠다드
	.NET 표준으로 개발하는 여러가지 영역에서 공통적으로 사용할 수 있는 클래스 라이브러리 프로젝트 중 하나이다.
	.NET 스탠다드로 만들게 되면 콘솔, 웹, 모바일, 윈도우와 같은 여러 환경에서 개발할 때, 
	스탠다드 프로젝트에 공통 코드를 만들 수 있다.

	.NET 스탠다드 기반의 클래스 라이브러리 프로젝트를 만들었다는 것은 .NET 모든 환경에서 참조하여 사용할 수 있음을 의미한다.

[3] 어셈블리
	클래스 라이브러리 프로젝트의 결과물은 DLL 파일이다.
	DLL 파일은 단독으로 실행할 수 없고, 어셈블리로 표현한다.
	어셈블리는 프로그래밍 학습용 자동차 세계에서는 교체 가능한 부품들을 의미한다.
	완성된 조각 하나의 기능을 모두 어셈블리로 볼 수 있다.

	어셈블리(프로젝트) > 네임스페이스 > 클래스
