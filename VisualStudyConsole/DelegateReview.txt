[1] 대리자
	대리자는 매개 변수 목록 및 반환 형식이 있는 메서드를 참조(포인터)를 나타내는 형식이다
	delegate는 위임하다, 대신하다는 의미가 있다.
	함수기능을 대신 호출하는 개념인 대리자 학습 고고우

[2] 대리자 특징
	대리자는 delegate 키워드 사용
	대리자는 함수 자체를 데이터 하나로 보고 의미 그대로 다른 메서드를 실행하는 기능이다.
	- 매서드의 매개변수로 대리자 변수를 넘길 수 있다.
	- 대리자를 사용하여 함수의 매개변수로 함수 자체를 전달할 수 있다.
	- 메서드의 매개변수로 또 다른 메서드 호출을 넘기는 기능이다.
	- 대리자는 동일한 메서드 시그니처를 갖는 메서드 참조를 담을 수 있는 그릇 역할을 한다.
	- 대리자는 람다와 같은 개념이다.
	- 대리자를 사용하면 함수를 모아 놓았다가 나중에 실행하거나 실행을 취소할 수 있다.
	- 대리자는 내부적으로 MulticastDelegate 클래스에서 기능을 상속한다.
	- 대리자는 이벤트를 만들어 내는 중간 단계의 키워드이다.

[3] 대리자 사용
	동일한 형태의 함수 (매개변수, 반환 타입)를 대신 호출 해줄 수 있도록 delegate 반환형 함수명() 형태로 구현이 가능하다.
	대리자의 첫 번째 목적은 이미 있는 함수를 대신 호출해주는 것이다.

	대리자 인스턴스 객체를 사용하여 메서드를 호출하는 방법 2개
	1) 인스턴스 객채(); // 직접 메서드 호출
	2) 인스턴스 객체.Invoke(); // 명시적인 호출

[4] 함수 포인터
	대리자 형식은 함수 포인터 (function pointer)라고도 한다.
	대리자를 생성할 때 사용되는 delegate 키워드는 이름이 없는 메서드를 만들 때 사용된다.

	대리자를 사용하는 것은 코드의 양을 증가시키는 것처럼 보이지만 이벤트를 만드는 필수 키워드이기에 배워둬야 한다.
	메서드와 이벤트의 중간에 위치하는 핼퍼 역할을 수행한다.

[5] 무명 메서드
	단어 그대로 이름이 없는 메서드이다.
